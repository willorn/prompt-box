# PromptBox 信息架构与分阶段实施方案

> 本文基于《[产品设计哲学](./PRODUCT_PHILOSOPHY.md)》制定，用于指导 PromptBox 的结构演进与逐步实现。

## 1. 目标信息架构

PromptBox 应逐步演进为 3 层结构：

### A. 调用层（默认入口）

这是用户最常进入的界面，目标是 **尽快找到并使用 Prompt**。

应包含：

- 搜索框
- 置顶快捷使用区
- 普通结果区
- 简洁详情区
- 立即使用动作

### B. 管理能力（就地完成）

目标是 **稳定维护 Prompt 资产**，但不单独切全局模式。

应包含：

- 新增
- 编辑
- 删除
- 置顶 / 取消置顶
- 分享（Markdown / JSON 复制）
- 复制内容 / 创建副本
- 分类管理
- 隐藏分类管理

### C. 系统层（设置 / 同步）

这是更低频的系统能力层，目标是 **配置环境，而不是频繁调用**。

应包含：

- 权限说明
- 快捷键说明
- 主题（跟随系统 / 浅色 / 深色）
- 登录时启动（后台待命）
- 导入 / 导出（JSON / Markdown / CSV，合并或替换；替换可撤销）
- WebDAV 同步
- 备份 / 恢复
- 快捷键与系统级设置

## 2. 当前状态与目标状态的差距

### 当前状态

- 主界面已统一为调用优先：单击 / 回车即用
- 管理动作通过右键、预览按钮、顶栏新增就地完成
- 同步 / 导入导出通过系统菜单进入

### 目标状态

- 列表主路径始终围绕“快速使用”
- 管理动作可用但视觉退后
- 系统能力继续下沉，不抢主流程

---

## 3. 分阶段实施策略

为避免一次性大改导致界面和逻辑失控，采用分阶段实现。

### 第一阶段：调用优先基础（已完成方向）

目标：

- 主界面统一为调用优先
- 管理动作就地可用，不单独切全局模式
- 新增、编辑、删除、置顶不抢主路径

完成标准：

- 单击 / 回车即用
- 右键与预览按钮承担管理
- 系统能力收在设置菜单

### 第二阶段：调用层瘦身

目标：

- 继续压缩默认调用界面的噪音
- 优化搜索、列表、详情区之间的职责边界

本阶段候选动作：

- 让详情区更聚焦“立即使用”
- 让普通卡片更像结果项而不是管理入口
- 优化置顶区与普通区的排序和视觉节奏

### 第三阶段：系统层下沉

目标：

- 把同步、导入导出、权限说明等能力更明确地收纳到系统层

本阶段候选动作：

- 重新组织右上角菜单
- 增加更清晰的系统说明入口
- 让调用页不承担系统级解释职责

### 第四阶段：使用智能化

目标：

- 让 PromptBox 越用越顺手，而不是越用越重

本阶段候选动作：

- 最近使用
- 使用频率排序
- 更优先的置顶策略
- 搜索优先级优化

## 4. 当前实现基线

当前产品已经收敛到：

1. 单一主界面，调用优先
2. 单击 / 回车 / ⌘1-9 快速使用
3. 右键与预览按钮管理；预览区提供「立即使用」主按钮
4. 预览可收窄 / 折叠，给列表更多空间
5. 默认列表：置顶 / 最近 / 全部；搜索时切到相关度单列表
6. 显示时始终置顶，并支持全屏应用上方唤起
7. call-path 冒烟测试锁定主路径符号，防止回归
8. 粘贴前切回唤起前的前台应用；多屏时跟随光标显示器
9. 搜索支持多词分词命中
10. 失焦自动隐藏（对话框期间抑制）；空结果可一键恢复
11. 主进程读写提示词统一 sanitize
12. 删除默认可撤销；原生对话框前抑制失焦隐藏
13. WebDAV 恢复可撤销；托盘交互抑制失焦误关
14. `bun run verify` 作为发布前检查入口
15. Shift 仅复制；内容变更自动本地安全快照
16. 再次唤起关闭残留弹层，保证干净调用态
17. 编辑弹层键盘保存；首次唤起权限轻提示
18. 非输入态 E/P/Delete(Backspace) 管理快捷键
19. WebDAV 恢复前强制安全快照；粘贴失败自动重试
20. 首次启动种子数据；Home/End 列表跳转
21. 非输入态直接打字即搜索（E/P 仅空搜索时管理）
22. 粘贴目标回退到上次成功应用；成功路径静默；⌘F 聚焦搜索
23. 搜索始终选中第一名；搜索框 Enter 优先当前选中
24. 缺权限轻量引导（每会话一次），不阻塞调用主路径
25. 带撤销 toast 抑制失焦隐藏；粘贴前自适应轮询自然前台；成功粘贴后恢复原剪贴板
26. 全角空格分词；⌘Backspace 清空搜索；安全快照菜单显示份数
27. 粘贴目标双路径激活；复制优先主进程剪贴板
28. 系统菜单统一入口；`bun run preflight` 发版预检
29. 系统菜单不挡调用快捷键；右键抑制失焦；renderer sandbox
30. 粘贴优先 hide 后自然前台；确认外部前台后再 Cmd+V；首次示例置顶
31. 唤起前 lsappinfo 快速识前台；粘贴激活优先 pid（校验进程名）
32. 全局快捷键冲突自动回退；会话目标记失败时清 pid
33. 空搜索数字键 1-9 一击即用；首次默认收起预览与侧栏（`]`）
34. 默认窗口更紧凑 860×600（最小 720×520），服务调用层密度
35. 编辑弹层取消复位；删除后选中相邻项
36. 每次唤起默认选中可见第一名；Enter 与数字键 1 对齐
37. 唤起后指针未有意移动前忽略悬停改选；移动后对准光标下卡片
38. 管理弹层锁定失焦隐藏；锁定期间 Alt+E/关闭不强制藏窗，保护未保存草稿
39. 再次唤起重置分类为「全部」；仅列表脏时重绘，缩短干净唤起
40. 顶栏「新增」「系统」在侧栏收起时仍可用
41. Esc 分层退场（toast → 清搜索 → 重置分类 → 隐藏）；清搜索回第一名
42. 预览收起时调用列表更密，优先选择速度
43. 有搜索词时 ⌫ 回改搜索；空搜索才删除选中（可撤销）
44. 序号角标在标题行；PageUp/PageDown 按页跳选
45. 唤起记前台 lsappinfo 合并读取，缩短显示等待
46. Tab/⇧Tab 搜索↔列表闭环；搜索结果回顶
47. 预览收起时分区更紧；置顶区视觉区分
48. 唤起记前台与窗口几何并行；粘贴成功异步保存
49. 粘贴成功静默保存串行队列，并立即标记列表脏
50. 仅空白视为空搜索；搜索框清空按钮
51. 粘贴后剪贴板约 2.8s 恢复；再次粘贴取消上次恢复
52. 未保存编辑时 focus-search/托盘唤起保护草稿，不静默清空
53. 系统菜单切换全局快捷键候选；回退时固定；不静默改偏好
54. 粘贴激活：pid 有效且非自己时不因显示名/进程名字符串不等丢掉

## 5. 实施约束

为保持产品稳定性，实施过程中遵守以下约束：

1. 不破坏现有快捷键主路径
2. 不降低“点击即用”的效率
3. 优先做结构分层，再做视觉细化
4. 每一步都必须在打包版中验证

---

## 6. 交付方式

后续改造采用以下循环：

1. 文档沉淀
2. 小步实现
3. 打包验证
4. 再进入下一步

这样可以确保 PromptBox 的演进始终沿着清晰的产品哲学前进，而不是在局部体验上不断来回摇摆。

---

## 7. 当前继续推进的实现点

基于前面的分层设计，当前继续往“调用优先”推进两件事：

### A. 执行优先交互

- 卡片是执行入口，不是纯预览入口
- 鼠标单击整张卡片即可使用
- Hover / 焦点负责预览，Click / Enter 负责执行
- Esc 应始终优先服务“退出当前层”
- 列表按置顶 / 最近 / 全部组织，服务肌肉记忆

### B. 使用智能化的第一步

- 记录最近使用时间与使用次数
- 在置顶区之外补一个“最近使用”层
- 让非置顶 Prompt 也能逐渐形成肌肉记忆路径
- 搜索结果按相关度排序，并综合使用频率与最近使用

这一步依然遵守原有约束：

- 不破坏 Alt+E 主路径
- 不牺牲键盘优先
- 不让管理动作重新抢回主界面
